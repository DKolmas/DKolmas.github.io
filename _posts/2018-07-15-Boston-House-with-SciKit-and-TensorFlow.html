Why using gradient descent optimization (TensorFlow) if closed-form solution exists (SciKit)¶

SciKit offers very easy to use way of training Linar Model. So I take the advantage of it to check what model I can get when Boston data set is used. SciKit LM model serves here as a quick jump in into a topic.

So why bother with TensorFlow when SciKit LM is available? There are two reasons:

  1. The first one is because I want to quickly check something I am interested in: how preprocessing of data affect training used with gradient descent optimization algoritm. For that I will use TensorBoard.
  2. The second reason is more of importance: when your data has many many features it is faster to use gradient descent optimization algorimth rather than closed-form eqation. I have preapared already fundamentals of gradient descent algoroithm here.
    SciKit learn Linear Regression model is a part of Generalized Linear Models. For Linear Model parameter estimation it uses closed-form solution:
    $\beta=(X^TX+I\lambda)^{-1}X^TY$
    With growing number of features simple linear model in SciKit becomes computationally inefficient comparing to gradient descent optimization algorithm. However there are many other optimization algorithms in SciKit that are mentioned in this post.

Becasue I use very small data set with small number of features only the first reason (data preprocessing impact on optimization outcome) is important to me know. And TensorFlow with Tensorboard suits here perfectly.

Implementaton of Linear Regresion using TensoFlow¶

In this part I use TensorFlow to create and train a linear model. Main focuse here is the check how data preprocessing affects result of training. First I am going to perform data preprocessing. And later on I will use oryginal data without preprocessing.

As for the preprocessing I choose to use standardization as it can help me in case of outlier happens in the data set. Using Min-Max preprocessing an outlier could destroy scaling of the feature (push most of the values into "corner" - very little value range) and make training difficult.
Without feature scaling (here: standardization) the gradinet descen algorithm would require more time to get the same result.

In [259]:
def feature_standardization(dataset):
    mu = np.mean(dataset,axis=0)
    sigma = np.std(dataset,axis=0)
    return (dataset - mu)/sigma
